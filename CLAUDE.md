# Project Guidelines

This is a Next.js + TypeScript + Tailwind CSS project.

## Auto-apply Skills

### UI / Component Development
When building or modifying UI components, pages, or layouts:
- Use `/frontend-design` for design quality and visual polish
- Use `/senior-frontend` for component architecture and Tailwind best practices
- Use `/baseline-ui` to validate animation durations, typography scale, and layout patterns

### React / Next.js
When writing or refactoring React/Next.js code:
- Use `/react-best-practices` to eliminate waterfalls and optimize rendering
- Use `/vercel-react-best-practices` for Vercel-specific performance patterns
- Use `/vercel-composition-patterns` when designing reusable or compound components

### Code Review / Refactoring
After completing a feature or significant change:
- Use `/simplify` to review changed code for quality and efficiency

### Accessibility
When adding interactive controls, forms, dialogs, or any user-facing UI:
- Use `/fixing-accessibility` to ensure ARIA labels, keyboard navigation, and WCAG compliance

### SEO / Metadata
When modifying page-level files or adding new routes:
- Use `/fixing-metadata` to ensure proper meta tags, OG tags, and structured data

### Animation / Motion
When adding transitions, animations, or scroll-linked effects:
- Use `/fixing-motion-performance` to prevent layout thrashing and ensure compositor-friendly properties

### UI/UX Design Decisions
When making design system decisions (colors, typography, spacing, component patterns):
- Use `/frontend-design` for distinctive, production-grade interface design
- Use `/ui-ux-pro-max` for design intelligence and style guidance
- Use `/web-design-guidelines` to review against best practices
